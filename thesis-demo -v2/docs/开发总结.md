## 1. 项目概述

本项目围绕 **Stage4 深度学习浮点数据压缩算法**，构建了一套完整的可视化演示与交互分析平台。系统支持：

- **地震数据压缩**：基于 CNN 因果预测 + 范围编码的无损 float32 压缩
- **算法交互演示**：逐步展示浮点位拆解、因果特征构建、CNN 概率预测、范围编码过程
- **通用压缩对比**：LZ77/RLE/Deflate/Neural Predictor 四种算法对比
- **SEG-Y 文件分析**：元数据解析、剖面热力图、单道波形、位统计分析

## 3. 模块详解

### 3.1 algorithm/ — 核心算法

**数据流**：
```
float32 数据 → extract_float_exponents() → uint8 指数体积
                    │
                    ▼
build_single_stage4_feature_causal_edge() → (C, H, W) 因果特征图
                    │
                    ▼
Small2DCNN → 256 类 logits → softmax → 概率分布
                    │
                    ▼
Stage4GlobalDiagonalRangeCodec
  ├── 按对角线扫描坐标
  ├── 批量预测 CDF
  └── RangeEncoder → .s4rc 位流
```

**关键类/函数**：

| 文件 | 类/函数 | 职责 |
|------|---------|------|
| `common.py` | `extract_float_exponents()` | float32 → uint8 指数（位操作提取） |
| `common.py` | `VolumeShape`, `VolumeData` | 三维数据形状封装 |
| `stage4.py` | `Small2DCNN` | 小型 2D 卷积网络（3 层 Conv + GAP + FC） |
| `stage4.py` | `build_single_stage4_feature_causal_edge()` | 构建单点 4/6 通道因果特征图 |
| `stage4.py` | `predictor_loco_i_2d()` | LOCO-I 预测器（左+上-左上，clamp[0,255]） |
| `codec.py` | `Stage4GlobalDiagonalRangeCodec` | 全局对角线编码器（推荐布局） |
| `range_coder.py` | `RangeEncoder`, `RangeDecoder` | 底层算术编码实现 |

**特征图 6 通道含义**（`diagonal_causal_edge` + `residual` 模式）：

| 通道 | 名称 | 值域 | 说明 |
|------|------|------|------|
| 0 | 像素值 | 0-255（÷255 归一化） | 因果可见区域的实际指数 |
| 1 | 可用掩码 | {0, 1} | 真实或映射的可用像素 |
| 2 | 因果掩码 | {0, 1} | 对角因果顺序下可见的像素 |
| 3 | 映射掩码 | {0, 1} | 从邻居"克隆"过来的像素 |
| 4 | 预测值 | 常量 | LOCO-I 预测值（全网格统一值） |
| 5 | 残差值 | 0-255（÷255 归一化） | (实际值 - 预测值) mod 256 |

### 3.2 visualizer/backend/ — FastAPI 后端

**设计原则**：
- 算法路径通过 `sys.path.insert` 动态引入 `algorithm/`，不修改原有代码
- SEG-Y 解析使用纯 Python 实现，不依赖 `segyio`（依赖复杂）
- 元数据缓存（`_sgy_meta_cache`、`_bit_stats_cache`）避免重复扫描
- IBM float → IEEE float 自动转换（热力图端点）

**sgy_parser.py SEG-Y 解析器**：

```
SEG-Y 文件结构：
┌──────────┬──────────┬─────────────────────────────────┐
│ 3200 B   │  400 B   │  Trace 0  │ Trace 1 │ ...      │
│ 文本头   │ 二进制头 │ 240B+数据  │ 240B+数据│          │
│ (EBCDIC) │ (参数)   │            │         │          │
└──────────┴──────────┴─────────────────────────────────┘

解析流程：
f.read(3200) → EBCDIC→ASCII 解码
f.read(400)  → 解析 sample_count, format_code, sample_interval
计算 trace_total = 240 + sample_count * bytes_per_sample(format_code)
计算 trace_count = (file_size - 3600) / trace_total
全文件扫描道头 → 收集 inline_3d / crossline_3d / SourceX / SourceY
推断维度 (profile_count × traces_per_profile)
```

### 阶段 5：SEG-Y 查看器开发

**问题**：需要独立页面查看 SEG-Y 地震数据，包含元数据解析、热力图、波形、位统计。

**解决**：
- 纯 Python SEG-Y 解析器（EBCDIC 解码、二进制头/道头解析、网格推断）
- Canvas 热力图（蓝-白-红光栅，2:1 宽高比）
- Canvas 波形图（单道振幅曲线 + 填充）
- numpy 全文件位统计（符号位/指数位/尾数位 23 bit-plane 分析）
- 250 行紧凑 JS，无 D3 依赖

### 阶段 6：性能优化

**问题**：全文件 4.2 亿样点扫描极慢（Python 逐点循环）。

**解决**：
- 内层 for s 循环全部替换为 `numpy.frombuffer` → `reshape(-1,4)` 向量操作
- `np.bincount` 替代 Python 累加计数
- `np.count_nonzero` 替代逐位判断
- 提速约 15 倍（从预估 10+ 分钟降至 50 秒）

## 5. 算法理解

### 5.1 IEEE754 浮点数结构

```
31  30      23  22                      0
┌────┬─────────┬──────────────────────────┐
│Sign│ Exponent │        Mantissa          │
│ 1b │   8b     │         23b              │
└────┴─────────┴──────────────────────────┘
value = (-1)^sign × 2^(exp-127) × (1 + mantissa/2^23)
```

### 5.2 Stage4 压缩原理

```
原始 float32 数据
        │
        ▼
  extract_float_exponents()       ← 提取 8-bit 指数
        │
        ▼
  build_stage4_features()         ← 构建因果特征图（对角线扫描顺序）
        │
        ▼
  Small2DCNN.forward()            ← 输出 256 类概率
        │
        ▼
  softmax → CDF                   ← 量化为范围编码的 CDF
        │
        ▼
  RangeEncoder.encode_symbols()   ← 按 CDF 编码真实符号
        │
        ▼
  .s4rc bitstream                 ← 压缩后的位流
```

**关键约束**：特征构建必须使用**因果关系**（解码时不能看到未来的像素），因此推荐 `diagonal_causal_edge` 模式。

### 5.3 LOCO-I 预测器

```
pred = clamp(left + up − up_left, 0, 255)
```

边界情况（左/上缺失时退化为单边预测或 0）。

### 5.4 残差模式

```
target_symbol = (real_exp − LOCO-I_prediction) mod 256
```

这使得预测目标集中在 0 附近，大幅降低熵。

### 5.5 范围编码

```
输入: CDF[256+1], symbol
流程:
  range = high - low + 1
  low  = low + range * CDF[symbol] / total
  high = low + range * CDF[symbol+1] / total - 1
重复直到所有符号编码完成 → 最终区间内任意整数 = 编码结果
```

### 5.6 IBM 浮点格式

```
IBM float (32-bit):
value = (-1)^sign × 16^(exp - 64) × mantissa / 2^24

与 IEEE754 的关键区别：基数为 16 而非 2
```

### 5.7 SEG-Y 格式

```
3200B 文本头 (EBCDIC) + 400B 二进制头 + 道数据
每道 = 240B 道头 + N_sample × 格式字节数 (通常 4B)
二进制头关键字段:
  bytes 16-18: sample_interval (μs)
  bytes 20-22: sample_count
  bytes 24-26: format_code (1=IBM, 5=IEEE)
道头关键字段:
  bytes 72-80: SourceX, SourceY
  bytes 188-196: INLINE_3D, CROSSLINE_3D
```

## 6. 代码规范

### 6.1 HTML 页面规范

- 所有页面包含统一的顶部导航栏（`.top-nav`）：Logo + 标签语 + `.nav-links` 链接组
- 导引页无侧边栏；功能页使用标准 `.main-container` > `.sidebar` + `.content-area` 布局
- 静态度资源引用使用 `?v=N` 版本号控制缓存
- Meta 标签包含 `Cache-Control: no-cache, no-store, must-revalidate`

### 6.2 CSS 规范

- CSS 变量统一在 `variables.css` 的 `:root` 中定义
- 类名采用 BEM 风格混合语义命名（`*-section`, `*-panel`, `*-card`, `*-grid`, `*.active`, `*.hidden`）
- 响应式断点：`@media (max-width: 1024px)` 和 `@media (max-width: 768px)`
- `.hidden { display: none !important }` 全局工具类

### 6.3 JavaScript 规范

- SEG-Y 查看器使用 ES5 语法（`var`, `function`, 字符串拼接），确保兼容性
- 其他页面使用 ES6+ 语法（`const`, `let`, 箭头函数, 模板字符串）
- 所有 DOM 操作前检查元素存在性：`if (btn) btn.addEventListener(...)`
- API 通信统一使用 `API_BASE_URL` 常量（可由 `?api=XX` URL 参数覆盖）
- 全局状态封装在 `AppState` 对象中
- 步骤导航封装在 `StepNavigation` 类中

### 6.4 Python 规范

- 算法模块导入使用 `sys.path.insert(0, str(ALGORITHM_DIR))` 动态路径
- 所有文件路径使用 `Path(__file__).resolve().parent` 推导
- 元数据缓存使用模块级字典（`_sgy_meta_cache`, `_bit_stats_cache`）
- 全文件扫描使用 numpy 向量化操作，避免 Python 逐元素循环

### 6.5 命名约定

| 类别 | 约定 | 示例 |
|------|------|------|
| CSS 类 | 功能-组件 (BEM-like) | `.sgy-heatmap-box`, `.sv-stats-card` |
| JS 函数 | 驼峰命名 | `renderCanvas()`, `loadHeatmap()` |
| JS 私有函数 | 下划线前缀 | `_heat()`, `_pie()`, `_drawBitPlane()` |
| Python 函数 | 下划线命名 | `parse_sgy()`, `_get_sgy_meta()` |
| DOM ID | 驼峰命名 | `sgyFileInput`, `heatmapContainer` |
| API 路径 | 小写连字符 | `/api/sgy/bit-stats` |

## 7. 关键技术决策

| 决策 | 原因 | 替代方案 |
|------|------|---------|
| Canvas 替代 SVG 热力图 | 120 万 DOM rect 渲染 5-10s → Canvas ImageData <100ms | D3.js SVG（已废弃） |

| D3 仅保留 demo.html | 波形、热力图用更快的 Canvas；CDF/概率图保留 D3 | 全 D3 或全 Canvas |

## 总结

本项目从最初混乱的单页应用逐步演化为结构清晰、模块独立、样式统一的多页面可视化平台。核心工作包括：

1. **架构重构**：项目目录模块化，消除硬编码路径
2. **页面拆分**：单页 → 5 页面（导引+压缩+演示+通用压缩+SEG-Y）
3. **浅色主题**：CSS 变量系统一键切换，6 个样式文件统一管理
4. **SEG-Y 工具集**：元数据解析、热力图、波形、位统计四大功能
5. **性能优化**：SVG → Canvas 渲染、Python 循环 → numpy 向量化

系统启动方式：
```bash
cd visualizer
python serve.py          # 启动后端 (8000) + 前端 (8080)
# 访问 http://localhost:8080
```
