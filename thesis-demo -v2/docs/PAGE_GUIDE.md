## 1. 系统概述

Stage4 是一套基于深度学习的**无损浮点数据压缩系统**，专为地震 SEG-Y 格式数据设计。系统前端包含四大功能页面：

| 页面 | 文件 | 用途 |
|------|------|------|
| 算法演示 | `demo.html` | 5 步 3D 可视化，分步展示压缩算法全流程 |
| 压缩工具 | `compress.html` | 上传 SGY → 压缩/解压 → 结果分析 |
| 通用压缩 | `general-compress.html` | 4 种经典算法对比（LZ77 / RLE / Deflate / Neural） |
| SEG-Y 查看器 | `sgy-viewer.html` | 元数据解析 / 剖面热力图 / 单道波形 / 位统计分析 |

## 2. 算法演示页（demo.html）

> 核心技术：Three.js 3D 渲染 + Canvas 2D 降级 + FastAPI 后端实时数据

用户上传 SGY 文件后，系统自动截取 `2×2×100` 小体积数据，借助 PyTorch Small2DCNN 模型（预加载并缓存全部 400 个坐标点的概率预测结果）实现**分步实时查询**——每一步向后端请求对应采样点的算法中间结果，3D 场景即时渲染。

页面布局：**左侧控制面板 + 中央 3D 画布 + 右侧原理指标面板 + 顶部步骤标签 + 底部前后翻页**。

### 2.1 步骤 1：Bit 拆解

**做什么**：展示当前采样点的 Float32 值按 IEEE 754 标准拆解为 32 个 Bit，直观理解浮点数在内存中的表示。

**3D 场景元素**：

| 元素 | 说明 |
|------|------|
| **32 个 3D 立方体** | 沿 X 轴一字排开，左到右对应 Bit 31 (MSB) → Bit 0 (LSB) |
| **分组括号线** | 三色括号分别框出 Sign / Exponent / Mantissa 三个位段 |
| **浮动数学公式** | `(-1)^s × 2^(e−127) × 1.m` |
| **大字号浮点值** | 当前采样点的科学计数法表示 |

**颜色系统**：

| 颜色 | 十六进制 | 含义 | 位数 |
|------|---------|------|------|
| 🔴 红色 | `#ef4444` | **符号位 (Sign)**：Bit 31。0 = 正数，1 = 负数 | 1 bit |
| 🟠 琥珀 | `#f59e0b` | **指数位 (Exponent)**：Bit 30~23。存储 biased exponent（实际指数 + 127） | 8 bits |
| 🔵 青色 | `#06b6d4` | **尾数位 (Mantissa)**：Bit 22~0。存储 fraction 部分（隐含 1.xxx） | 23 bits |

**明暗逻辑**：
- **亮立方体（高发光）**：该 bit = 1，立方体具有颜色发光（emissive intensity = 0.85）
- **暗立方体（低不透明）**：该 bit = 0，立方体半透明（opacity = 0.45），仅有弱发光

**交互**：
- 点击任意立方体 → 控制台输出该 Bit 的详细信息
- 悬停任意立方体 → 立方体放大 1.35×，产生脉冲波纹扩散到周围

**算法含义**：Stage4 只对 **8 个指数位**做 CNN 预测压缩。符号位（1bit）高度随机、尾数位（23bit）接近均匀噪声，两者均用 zlib 通用压缩器处理。这种分解策略是 Stage4 的核心设计思想——将"可预测"的指数与"不可预测"的符号/尾数分离处理。

### 2.2 步骤 2：特征提取

**做什么**：展示**因果约束（Causal Constraint）**——编码当前采样点时，只能使用空间上"已经编码过的"邻居作为 CNN 输入上下文。

**3D 场景元素**：

| 元素 | 说明 |
|------|------|
| **10×10×10 体素网格** | 模拟三维数据空间（profile × trace × sample），每个体素代表一个采样点 |
| **对角扫描线** | 粉红色平面沿 diagonal 顺序推进，模拟编码/解码时的扫描路径 |
| **当前点球体** | 粉色球体标记正在编码/解码的位置，带呼吸脉冲动画 |
| **17×17 Causal Patch 平面** | 右侧浮动的半透明平面，展示二维 profile-trace 平面上的因果 Patch |

**颜色系统**：

| 颜色 | 十六进制 | 含义 |
|------|---------|------|
| 🔵 青色 | `#06b6d4` | **已编码体素 (Causal)**——扫描线已走过，数据已知，可作为 CNN 上下文。中等不透明度（0.55）|
| 🔴 粉色 | `#ec4899` | **当前体素**——正在编码/解码的点。最高不透明度（0.95），强发光 |
| ⚪ 灰色 | `#9ca3af` | **未编码体素 (Future)**——扫描线未到达，数据未知，不能作为上下文。低不透明度（0.40）|

**Patch 平面**：
- 粉色中心格：当前采样点位置（17×17 的中心，坐标为 (8, 8)）
- 青色格子（0.55–0.85 不透明度）：**因果可用区域**——已编码的点，特征被填入 CNN
- 灰色格子（0.28 不透明度）：**非因果区域**——未编码的点，特征填 0

**算法含义**：编码器 (Encoder) 和解码器 (Decoder) 必须在完全相同的信息下工作。如果 CNN 的输入包含了尚未编码的邻居的信息，解码器将无法复现编码器的预测，导致解码失败。对角线扫描顺序（diagonal order）保证了在 profile-trace 平面上，当前位置所有"上游"邻居都已被处理。

**Causal Patch 示意**（9×17 为例）：
```
┌─────────────────┐
│ ░ ░ ░ ░ ░ ░ ░ ░ │  ← 非因果区域（未编码，填 0）
│ ░ ░ ░ ░ ░ ░ ░ ░ │
│ ░ ░ ░ ░ ▓ ▓ ▓ ▓ │  ← 对角线边界
│ ▓ ▓ ▓ ▓ ■ ░ ░ ░ │  ← ■ = 当前点
│ ▓ ▓ ▓ ▓ ▓ ░ ░ ░ │  ← ▓ = 因果区域（已编码，填入真实值）
└─────────────────┘
```

### 2.3 步骤 3：CNN 预测

**做什么**：Small2DCNN 接收因果 Patch（17×17×C 的特征张量），输出 256 类的概率分布（对应指数值 0~255 各有多大概率）。

**3D 场景元素**：

| 元素 | 说明 |
|------|------|
| **256 根概率柱** | 沿弧形排列，柱高度与该符号的预测概率成正比 |
| **概率曲面（Tube）** | 连接柱顶的平滑管道曲面，展示整体概率分布形状 |
| **粉色 Beacon 光柱** | 标记真实 Symbol 位置的垂直光柱，从圆顶射下 |

**颜色系统**：

| 颜色 | 含义 |
|------|------|
| 🌈 **彩虹连续色阶** | 256 个符号的概率柱，色调从 HSL(0) 红 → HSL(0.33) 绿 → HSL(0.66) 蓝 → HSL(1) 红（闭合），连续映射 0→255 |
| 🔴 粉色 | `#ec4899`：**真实 Symbol** 的 Beacon 光柱和中心大字标注（"Symbol X"） |
| 🔵 靛蓝 | `#5b6af0`：概率柱顶部的连接曲面 |
| ⚪ 淡蓝 | `#667eea`：数据流管道（粒子从 CNN 流向中心再散射到概率柱） |

**左侧 CNN 结构模型**（层叠方块）：

```
┌──────────────────┐
│ Conv2D 3×3       │  青色 0x06b6d4
├──────────────────┤
│ ReLU             │  深青 0x0891b2
├──────────────────┤
│ Conv2D 3×3       │  蓝灰 0x0e7490
├──────────────────┤
│ Conv2D 3×3       │  深蓝 0x155e75
├──────────────────┤
│ GAP → FC 256     │  靛蓝 0x1e3a8a (输出 256 类)
└──────────────────┘
```

**周围粒子**：靛蓝粒子从 CNN 向左散射再汇聚到概率柱区域，模拟数据流动。

**算法含义**：CNN 输出 256 维 softmax 概率。预测越精准 → 真实符号的概率值越高 → Range Coder 需要的信息越少 → 压缩率越好。理想情况下，如果所有预测 100% 准确（概率 = 1.0），每个指数位只需 0 bit（完全确定）。

### 2.4 步骤 4：Range Coding

**做什么**：将 CNN 概率分布量化为 CDF，Range Encoder 根据符号的概率区间递归缩窄编码范围，输出 bitstream。

**3D 场景元素**：

| 元素 | 说明 |
|------|------|
| **主概率条（Main Bar）** | 13 单位宽的 256 段彩色条，每段宽度 = 该符号的概率。粉色箭头指向当前编码的 Symbol |
| **三层递归隧道** | 从大到小的三层概率条，模拟 Range 编码的区间逐层缩窄：`[0,1) → [cdf_low, cdf_high)` |
| **Bitstream 方块** | 底部 20 个 3D 方块，bit=1 为绿色 `#10b981`（发光），bit=0 为灰色 `#9ca3af` |

**主概率条颜色**：

| 颜色 | 含义 |
|------|------|
| 🌈 HSL 连续色阶 | 256 个符号对应的概率区间，颜色连续过渡 |
| 🔴 粉色 `#ec4899` | 当前编码 Symbol 的区间段——唯一具有 emissive glow 的段（emissiveIntensity = 0.9） |
| ⚫ 灰色 `#475569` | 概率条的线框边框 |

**隧道颜色**：

| 层级 | 说明 |
|------|------|
| 第 1 层 | 基于全概率分布的区间，Symbol 段被放大突出 |
| 第 2 层 | 缩窄到 symbol 的概率区间内部（scale = 0.6） |
| 第 3 层 | 再次缩窄（scale = 0.36），模拟多次递归 |

**公式标注**（右侧 3D 文字）：
```
range = high - low + 1
low  += range × cdf[symbol]
range = range × prob[symbol]
```

**实测数据横幅**（仅真实数据时，左下角绿色文字）：
```
Range: [0.4231, 0.4285)
Prob: 0.54%
Bits: 1624  (156/400)
```
每项含义：当前编码区间 → 当前符号概率 → 累计输出 bit 数及进度。

**Bitstream 方块**：

| 颜色 | 含义 |
|------|------|
| 🟢 绿色 `#10b981` | bit = 1，带发光效果 |
| ⚪ 灰色 `#9ca3af` | bit = 0 |

**算法含义**：Range Coder 是算术编码 (Arithmetic Coding) 的整数实现。它用概率区间宽度决定编码长度——宽区间（高概率）→ 缩窄后需要信息少 → 编码短；窄区间（低概率）→ 缩窄后相对信息量大 → 编码长。这就是为什么 CNN 预测越准、压缩率越好的根本原因。

## 3. 压缩工具页（compress.html）

> 核心流程：上传 → 压缩 → 结果分析。支持压缩/解压双模式。

### 3.1 功能概览

| 步骤 | 内容 |
|------|------|
| **步骤 1：数据上传** | 拖拽或点击上传 .sgy / .segy / .bin / .dat 文件。支持压缩/解压模式切换 |
| **步骤 2：执行压缩** | 配置 CNN 参数（特征模式、目标模式、Patch 大小、批次大小、计算设备），点击开始 |
| **步骤 3：结果分析** | 查看压缩统计、下载结果、解压验证 |

### 3.2 压缩参数配置

| 参数 | 可选值 | 说明 |
|------|--------|------|
| 特征模式 | 对角因果边缘 / 因果边缘 | CNN 输入特征的提取方式，推荐"对角因果边缘" |
| 目标模式 | 残差值 / 原始值 | 预测目标——相邻指数差（residual）或原始指数值（raw），需与 checkpoint 匹配 |
| Patch 高度 | 3–21（奇数） | 因果 Patch 在 profile-trace 平面上的行数，推荐 9 |
| Patch 宽度 | 3–33（奇数） | 因果 Patch 的列数，推荐 17 |
| 批次大小 | 1–32768 | GPU 推荐 4096–16384，CPU 推荐 1 |
| 计算设备 | GPU(CUDA) / CPU | GPU 可提速 10–50 倍 |

### 3.3 压缩流程

```
SGY 上传
  ↓
SEG-Y 头文件提取（EBCDIC 文本头 + 二进制头 + 道头）
  ↓
Float32 数据提取（IBM→IEEE 自动转换）
  ↓
三通道分解：
  ├── signs   = (u32 >> 31) & 0x1   → zlib 压缩 → sign.zlib
  ├── mants   = u32 & 0x7FFFFF       → zlib 压缩 → mant.zlib
  └── exps    = (u32 >> 23) & 0xFF   → Stage4 CNN + Range Coder → compressed.s4rc
  ↓
合并统计：total = .s4rc + sign.zlib + mant.zlib
  ↓
压缩比 = 原始大小 / total
```

### 3.4 解压验证

压缩完成后点击"解压验证"，后端将：
1. 从 `.s4rc` 中 CNN 解码指数
2. 从 `sign.zlib` / `mant.zlib` 解压符号和尾数
3. 重建 Float32 数据
4. 用原始 SGY 头文件重建完整 SGY
5. 逐元素比对，输出 MSE 和 SHA256 验证结果

### 3.5 独立解压

切换到"解压"模式，上传压缩产出的文件：
- `.s4rc`（必选）

## 4. 通用压缩页（general-compress.html）

> 对比 4 种经典压缩算法，展示 Stage4 算法与通用压缩器在不同数据类型上的差异。

### 4.1 支持的算法

| 算法 | 类型 | 核心原理 |
|------|------|---------|
| **LZ77 + Huffman** | 字典编码 + 熵编码 | 滑动窗口查找重复模式 → Huffman 编码，gzip/zip 核心 |
| **RLE 行程编码** | 行程编码 | 连续重复数据用 (值, 长度) 对表示，字节 255 作哨兵 |
| **Deflate** | 标准库实现 | Python zlib 实现，PNG/ZIP/gzip 标准，LZ77 + Huffman |
| **Neural Predictor** | 神经网络 | MLP 预测下一字节 → 残差编码 |

### 4.2 两种模式

**文本对比模式**：
- 输入任意文本 → 4 种算法同时压缩 → 横向对比压缩比
- 显示每条算法的：压缩后大小、压缩比、节省空间百分比、验证结果、耗时
- 最佳算法标记 🏆 徽章
- 建议测试文本：重复文本（RLE 擅长）、模式重复（LZ77 擅长）、自然语言（Deflate 擅长）、有规律数列（Neural 擅长）

**文件压缩模式**：
- 上传任意文件 → 选择一种算法 → 压缩
- 显示压缩统计并提供下载链接

### 4.3 算法特性对比表

| 算法 | 类型 | 适用场景 | 压缩速度 | 压缩比 |
|------|------|---------|---------|--------|
| LZ77 + Huffman | 字典+熵编码 | 通用文本、代码 | ★★★★ | ★★★★ |
| RLE | 行程编码 | 图像、重复数据 | ★★★★★ | ★★ |
| Deflate | 标准算法 | 通用压缩、归档 | ★★★★★ | ★★★★★ |
| Neural Predictor | 神经网络 | 模式数据、序列 | ★★★ | ★★★★ |

## 5. SEG-Y 查看器（sgy-viewer.html）

> SEG-Y 格式地震数据的全流程分析与可视化工具。

### 5.1 功能概览

| 步骤 | 内容 |
|------|------|
| **步骤 1：文件解析** | 上传 .sgy 文件，自动解析 EBCDIC 文本头、二进制头、所有道头 → 显示元数据表格 |
| **步骤 2：剖面热力图** | 选择 Inline 剖面索引 → 渲染二维 Canvas 热力图（蓝-白-红 色阶） |
| **步骤 3：单道波形** | 点击热力图道号或输入道索引 → 绘制振幅-采样点 Canvas 折线图 |
| **步骤 4：位统计分析** | 全文件扫描，分三个标签展示：指数位直方图+熵值、符号位饼图+空间分布、尾数位平面熵 |

### 5.2 步骤详解

#### 5.2.1 文件解析

解析的元数据包括：

| 字段 | 说明 |
|------|------|
| 文件大小 | MB |
| 数据格式 | IEEE Float / IBM Float / 整型 |
| 道数 | 总地震道数 |
| 每道采样点 | 每道的深度/时间采样点数 |
| 采样间隔 | μs（微秒） |
| 每道字节 | 道头 + 采样数据总字节 |
| 剖面数 | 推断的 Inline 剖面数 |
| 每剖面道数 | 推断的 Crossline / 道/剖面 |
| Inline 范围 | 道头中的 Inline 编号范围 |
| Crossline 范围 | 道头中的 Crossline 编号范围 |

#### 5.2.2 剖面热力图

- **渲染方式**：Canvas 像素级热力图，每道（宽 2px）× 每采样点（高 1px）
- **色阶**：蓝 `rgb(33,62,181)` → 青 → 绿 → 黄 → 红 `rgb(255,0,0)`（发散型色阶，中心为白）
- **色阶条**：右侧垂直色条，标注最大值 / 最小值
- **交互**：
  - 鼠标悬停：浮动 tooltip 显示道号、采样点、具体振幅值
  - 点击任意道：自动跳转到单道波形并绘制该道
- **信息栏**：剖面号 | 道数 × 采样数 | 值域范围 | 画布比例

#### 5.2.3 单道波形

- **渲染方式**：Canvas 折线图 + 半透明填充
- **线条颜色**：系统主色 `#5b6af0`，填充区 `rgba(91,106,240,0.12)`
- **背景网格**：浅灰 `#e8e8ec` 网格线
- **零值线**：灰色 `#ccc` 水平参考线
- **坐标轴**：X 轴=采样点，Y 轴=振幅值（自动缩放）
- **信息栏**：道号 | 采样点数 | 振幅值域

#### 5.2.4 位统计分析

三个子标签，全文件向量化扫描：

**指数位标签**：

| 图表 | 说明 |
|------|------|
| 指标卡 | 经验熵（bits）、理论下界（bits）、非零指数比例（%）、总样本数 |
| 直方图 | 256 bin 柱状图，X 轴 0–255 指数值，Y 轴频数，蓝色 `#5b6af0` 柱 |
| 均值空间热力图 | Inline × Crossline 的指数均值分布，同剖面热力图色阶 |

**符号位标签**：

| 图表 | 说明 |
|------|------|
| 指标卡 | 正数比例（%）、负数比例（%）、零值比例（%）、总样本数 |
| 饼图 | Canvas 饼图三色：蓝=正数 `#5b6af0`，红=负数 `#ef4444`，灰=零值 `#d1d5db` |
| 符号空间热力图 | Inline × Crossline 的符号倾向分布（蓝=正→白→红=负） |

**尾数位标签**：

| 图表 | 说明 |
|------|------|
| 指标卡 | 尾数总熵（bits）、理论上界 23 bits、熵占比（%）、总样本数 |
| 分组熵占比 | 高位 1–7 bit、中位 8–15 bit、低位 16–23 bit 各占比 |
| **位平面熵图** | 核心图——双 Y 轴 Canvas 图表：左轴蓝色柱 = 每 bit 熵值；右轴红线 = 累计熵；背景三色区域标注高位/中位/低位分组 |

**位平面熵图颜色**：

| 颜色 | 含义 |
|------|------|
| 🔵 蓝色柱 `#5b6af0` | 每 bit 熵值——高位熵低（有规律）、低位熵接近 1（随机） |
| 🔴 红色折线 `#ef4444` | 累计熵——23 个 bit 的熵值累计总和 |
| 🟣 蓝紫背景 | 高位 1–7 bit 区域，`rgba(91,106,240,0.06)` |
| 🟢 绿色背景 | 中位 8–15 bit 区域，`rgba(16,185,129,0.06)` |
| 🟠 橙色背景 | 低位 16–23 bit 区域，`rgba(245,158,11,0.06)` |

**关键发现**：高位 bit 熵值极低（≈0.01）→ 接近确定 → **可预测**；低位 bit 熵值接近 1.0 → 完全随机 → **不可预测**。这正是 Stage4 只压缩指数位（8 bits）而不压缩尾数位（23 bits × 高位不同熵分布）的数据依据。
